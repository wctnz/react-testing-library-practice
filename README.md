# React testing library практика  

React testing library — библиотека для тестирования реакт компонентов, позволяет создавать тестировочные кейсы максимально похожие на взаимодействие пользователя с интерфейсом. React testing library предоставляет доступ к дом-узлам и тестирует конечную разметку (а не внутреннее состояние компонента), что дает возможность писать более стабильные тесты.   

В практике разобраны     

- Поисковые типы  
- Поисковые варианты и их отличия друг от друга  
- Assertive functions (утверждающие функции)  
- Тестирование событий  
- Тестирование асинхронного кода  
- Тестирование библиотеки React-router, хуков useParams, withRouter, использование функции renderWithRouter  
- Тестирование библиотеки Redux, использование функции renderWithRedux  
- Тестирование хука useContext.   

Функции describe, test, it, render, expect, fireEvent, userEvent, mock, объект screen (содержит полную разметку приложения)  

Поисковые типы - различные типы (методы) выбора элементов (getByText, getByRole и так далее) — после того, как библиотека рендерит ваш компонент, она предоставляет большой набор методов для того, чтобы найти нужные элементы внутри разметки. Кроме стандартных способов обращения к элементам дом-дерева React testing library предоставляет метод getByTestId, который ищет элементы по специальному атрибуту data-testid  

Поисковые варианты  - способы нахождения различных элементов внутри дом-дерева. В отличие от поисковых типов, существуют также и варианты поиска — getBy, queryBy и findBy. Все расширяются с помощью типов поиска. getBy — нужно найти существующий элемент.  queryBy — нужно показать что элемента нет в дом.  findBy — изначально элемента не было, но при работе асинхронного кода он появится. Если элементов несколько нужно использовать AllBy  

Assertive Functions - утверждения, утверждающие функции (toBeNull, toBeInTheDocument и так далее) -  утверждающие функции выполняются справа от утверждения, у каждой из них свой функционал — проверяют наличие элемента в дом, проверяют наличие класса у элемента, есть ли на элементе какие то стили. Если необходимо получить отрицание, нужно добавить частица not  

fireEvent, userEvent — позволяют тестировать события, имитировать взаимодействие пользователей с интерфейсом. В чем отличие от fireEvent, userEvent имитирует реальное поведение браузера более точно.
